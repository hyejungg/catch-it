# QA Checklist: MVP

## 목적
- CatchIt MVP의 핵심 플로우/리스크 항목을 수동 QA로 점검한다.

## 환경
- Chrome 최신 버전
- 확장 프로그램 빌드 후 로드(`dist`)
- 테스트 페이지(일반 웹사이트 2개 이상)
- Notion 연동 테스트용 Token / Database ID

## 필수 체크 항목

### 1) 드래그 수집
- 텍스트 3자 이상 드래그 시 팝오버(저장/취소)가 표시된다.
- 3자 미만 드래그 시 팝오버가 표시되지 않는다.
- `Alt + Drag` 옵션 ON 상태에서는 Alt 없이 드래그 시 팝오버가 표시되지 않는다.
- 저장 클릭 시 항목이 Dashboard 목록에 추가된다.
- 취소 클릭 시 저장되지 않는다.

### 2) 대시보드 목록/검색
- 저장된 항목이 최신순으로 표시된다.
- 검색어로 `text/title/url/tag` 포함 검색이 동작한다.
- 복사 버튼이 텍스트를 클립보드에 복사한다.
- 원문 열기 버튼이 새 탭으로 URL을 연다.
- 삭제 버튼이 항목을 제거하고 목록에 즉시 반영된다.
- 확장 아이콘으로 열리는 UI 폭이 충분해 주요 컨텐츠가 잘리지 않는다.
  - Side Panel 모드 사용 시: 기본 진입이 side panel에서 동작한다.
  - Popup 모드 유지 시: 텍스트/버튼/상태 배지가 겹치지 않는다.

### 3) Settings 저장
- `notionToken`, `notionDbId` 입력값이 저장/재진입 후 유지된다.
- `autoSync`, `requireAlt` 토글이 저장/재진입 후 유지된다.
- 타이틀 구조가 요구사항과 일치한다.
  - 메인 타이틀: `CatchIt - 웹 텍스트 수집기`
  - 서브 타이틀: `Dashboard` / `Settings`

### 4) Notion 동기화
- `autoSync` ON 상태에서 저장 시 Notion 페이지가 생성된다.
- `Sync Now` 클릭 시 미동기화 항목이 배치 동기화된다.
- 성공 시 `notion.status = synced`, `pageId`, `syncedAt`이 반영된다.
- 설정 누락 시 `failed` 상태와 오류 메시지가 기록된다.
- 401/403/429 응답 시 오류 메시지가 구분되어 기록된다.
- `연동 확인(Test Connection)` 버튼으로 설정값 유효성을 즉시 확인할 수 있다.
- `Notion DB 바로가기(Open Notion)` 버튼으로 대상 데이터베이스 페이지가 열린다.
- `연동 확인` 실행 후 성공/실패 상태 메시지가 UI에 표시된다.

### 5) 상태 갱신/반영
- content-script 저장 후 Dashboard가 새로고침 없이 갱신된다.
- Sync 상태 변경이 목록 상태 배지에 반영된다.

## 회귀 리스크 체크
- 토큰/DB ID가 콘솔 로그에 노출되지 않는다.
- 팝오버가 페이지 클릭 시 정상적으로 닫힌다.
- 대량(예: 100개+) 데이터에서도 목록/검색 응답이 과도하게 느리지 않다.
